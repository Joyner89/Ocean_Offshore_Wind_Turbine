# Repository Guidelines

## Project Structure & Module Organization

This is a TypeScript Babylon.js WebGPU ocean demo. Application entrypoints live in `src/index.ts` and `src/createScene.ts`. Scene code is under `src/scenes/`; `ocean.ts` wires the main scene, while `wavesGenerator.ts`, `wavesCascade.ts`, `fft.ts`, and `initialSpectrum.ts` handle GPU wave simulation. Shared scene utilities are in `src/scenes/tools/`. Runtime assets are under `src/assets/`, including WGSL shaders and GLB/HDR/EXR resources. Project notes and plans can live in `dos/`. Build output is generated in `dist/` and should not be committed.

## Build, Test, and Development Commands

- `npm install`: install dependencies. If local npm cache permissions fail, use `npm install --cache /private/tmp/ocean-demo-npm-cache`.
- `npm start`: run `webpack-dev-server` on port `8080`. WebGPU works best over HTTPS; run `node scripts/ensure-https-cert.js` first if `certs/` is missing.
- `CHOKIDAR_USEPOLLING=true WATCHPACK_POLLING=true npm start`: use this on macOS if file watching fails with `EMFILE`.
- `npm run build`: create a production bundle in `dist/`.
- `npm run lint`: run ESLint over `src/**/*.{ts,tsx}`.

There is currently no `npm test` script or configured test framework.

## Coding Style & Naming Conventions

Use TypeScript with strict compiler settings from `tsconfig.json`. Follow `.editorconfig`: UTF-8, spaces, 4-space indentation. Keep class names in `PascalCase`, variables/functions in `camelCase`, and private class fields prefixed with `_`, matching existing scene classes. Keep Babylon.js 5.28.0 dependencies pinned unless compatibility is explicitly tested; this project includes WebGPU workarounds for newer browsers.

## Testing Guidelines

For now, validate changes with `npm run build` and `npm run lint`. For rendering changes, also run the dev server and verify `https://localhost:8080/` in a WebGPU-capable Chrome or Edge browser. When adding future tests, place them near the related module or in a dedicated `tests/` folder, and name files after the unit under test, for example `wavesGenerator.test.ts`.

## Commit & Pull Request Guidelines

Git history currently uses a simple imperative style, for example `Initialize OceanDemo WebGPU project`. Keep commits concise and action-oriented, such as `Add turbine telemetry client`. Pull requests should include a short summary, manual verification steps, browser/WebGPU notes, and screenshots or screen recordings for visual scene changes. Link related issues or planning documents when available.

## Security & Configuration Tips

Do not commit `certs/`, `.env*`, `dist/`, or `node_modules/`. Local HTTPS certificates are generated by `scripts/ensure-https-cert.js`. Treat external WebSocket endpoints and telemetry data as runtime configuration, preferably via URL parameters or environment-specific documentation rather than hardcoded secrets.
